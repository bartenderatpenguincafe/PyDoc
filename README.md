# PyDock (Docklight-like, COM only)

PyDock — лёгкий терминал для работы с COM-портом в стиле Docklight: проекты, наборы команд отправки, реакции на приём, журнал обмена, заметки по проекту.  
Стек: **Python 3.13 + PySide6**, Windows-ориентировано.

---

## 1) Быстрый старт (запуск из исходников)

1) Установите **Python 3.13** (желательно вместе с `py launcher`).

2) В корне проекта:
- создать виртуальное окружение и поставить зависимости:
  - `py -3.13 -m venv .venv`
  - `.venv\Scripts\activate`
  - `pip install -r requirements.txt`

3) Запуск:
- `python main.py`

---

## 2) Сборка в EXE

Сборка делается скриптом:

- `build\build.bat`

Скрипт:
- проверяет наличие **Python 3.13**,
- создаёт/использует `.venv`,
- проверяет/ставит **PySide6** и **PyInstaller**,
- собирает приложение в portable one-folder.

Где будет результат:
- **EXE**: `build\dist\PyDock\PyDock.exe`
- Папка сборки: `build\dist\PyDock\` (рядом лежат Qt-плагины/DLL и ресурсы)

---

## 3) Структура проекта

- `main.py` — точка входа (загрузка настроек, применение темы/шрифта, запуск UI)
- `app/` — код приложения
  - `ui/` — интерфейс (главное окно, диалоги, таблицы)
  - `core/` — модель проекта, настройки приложения, логика движка
  - `transports/` — транспорт COM (QtSerialPort)
  - `util/` — утилиты (пути к ресурсам, dark titlebar для Windows)
  - `i18n.py` — переводы RU/EN
- `resources/` — темы и ресурсы (QSS и прочее)
- `build/` — скрипты/артефакты сборки

---

## 4) Что такое “проект” и где он хранится

Проект — это один JSON-файл:
- расширение: `*.pydock.json`

Проект содержит:
- настройки COM-порта (port/baud/parity/stopbits/flow…)
- список отправляемых команд (Send)
- список правил приёма (Receive: wildcard-шаблоны + действие)
- текст “Заметки по проекту”

---

## 5) Настройки приложения

Открываются кнопкой **“Настройки приложения”** на панели.

Там:
- **Язык**: Русский / English
- **Тема**: `light` / `dark`
- **Шрифт** и размер
- Цвета:
  - RX (принято) — по умолчанию красный
  - TX (отправлено) — по умолчанию синий
  - SYS/прочее — по умолчанию серый
- “Компактный интерфейс”

Где хранятся:
- `%APPDATA%\PyDock\settings.json`

---

## 6) Настройки проекта (COM)

Открываются кнопкой **“Настройки проекта”** на панели.

Параметры:
- COM-порт
- Baud
- Data bits
- Parity
- Stop bits
- Flow control
- Byte order (LSB/MSB) — предпочтение проекта (на текущем этапе влияет как “настройка”, логика интерпретации может расширяться дальше)

---

## 7) Отправка команд (Send)

Слева сверху таблица отправки.

- Каждая строка — команда/последовательность отправки.
- Кнопка **➜** в первой колонке строки — отправить эту команду.
- Также можно отправлять выбранную строку клавишей **Space**.
- Двойной клик по строке — редактирование.
- Последняя строка `<двойной клик: добавить>` — быстрое добавление новой команды.

Редактирование Send:
- Имя
- Формат: `hex` или `ascii`
- Данные
- (если включено) контрольная сумма (checksum)

---

## 8) Приём и реакции (Receive)

Слева снизу таблица приёма.

- Каждая строка — правило, которое проверяется по входящим байтам.
- Используется **wildcard** `??` (байт-джокер) в HEX-шаблонах.  
  Пример: `FF 52 ??` — матчится на `FF 52 01`, `FF 52 02` и т.д.

Действия (Action):
- `none` — ничего
- `log` — запись комментария в лог (поддерживает макрос `%HEX%`)
- `send` — авто-ответ: отправить одну из команд Send по ID

Редактирование Receive:
- Active (вкл/выкл)
- Name
- Pattern (HEX, допускается `??`)
- Action type
- Text для log
- Send ID для send

---

## 9) Журнал (Log)

Справа сверху — журнал обмена.

Цвета по умолчанию:
- RX — красный
- TX — синий
- SYS/прочее — серый

Цвета настраиваются в настройках приложения.

---

## 10) Заметки по проекту

Справа снизу — поле заметок.  
Сохраняется внутри файла проекта `*.pydock.json`.

---

## 11) Полезные замечания

- Приложение ориентировано под Windows (в тёмной теме включается тёмный заголовок окна через DWM, если поддерживается системой).
- Если при сборке EXE на чистой машине приложение не стартует — чаще всего причина в Qt-плагинах. Скрипт сборки уже использует `--collect-all PySide6`, что закрывает большинство типовых проблем.
